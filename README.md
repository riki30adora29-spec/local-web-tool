# local-web-tool

一个**模式目录**，不是 npm 包，不是框架。帮你从零搭一个跑在本地的单页网页工具（Node.js + Express + 浏览器打开即用）。

## 这能干什么

想做一个小工具——调 AI 帮你写文案、分析内容、整理数据——但不想搞数据库、不想部署服务器、不想折腾登录注册。只想双击一个图标，浏览器打开，用完关掉。

这个项目把这条路每一步踩过的坑全标出来了。

## 里面有什么

- **SKILL.md** — 主文档：架构、服务骨架、API 代理模式、LLM prompt 模板
- **Pitfalls（踩坑合集）** — WSL2 连不上、CIFS 启动要 30 秒、.bat 被浏览器劫持、双击 index.html 调不了 API…每一个都有症状→原因→修复
- **templates/** — 复制就能用的启动/停止脚本
- **references/** — 深度专题：磨砂玻璃、极光背景、字体搭配、侧边栏、主题迁移、i18n、测试流程…

## 和框架有什么区别

框架给你一套规矩让你遵守。这个项目给你一堆"别人已经踩过的坑"和"复制就能跑的模板"，你怎么组合是你的事。

## 怎么用

1. 看 `SKILL.md`，对着搭一个最小原型
2. 遇到报错，查 Pitfalls 那一节——大概率已经在里面了
3. 需要视觉效果，去 `references/` 里翻对应的专题
4. 调通之后，`templates/start.bat` 拿来改一改就能双击启动

## 适用环境

- WSL2 + Windows（主要）
- 原生 Windows Node.js（见 `references/windows-native.md`）
- macOS / Linux 用户需要自己改启动脚本（服务逻辑完全通用）
